# Feishu Collaboration Guide

## Group Naming

Use one Feishu group per project:

- `[项目简称]-产品研发群`
- `[项目简称]-版本迭代群` for major later iterations if needed

## Members

Each project group should include:

- Product Manager
- Project Manager
- Test Engineer
- Backend Engineer
- Frontend Engineer

## Pinned Messages

Pin these messages in every project group:

1. project goal and success metric
2. current version scope
3. project folder path
4. latest release / rollback status
5. requirement change process

## Message Discipline

- Requirement changes must include effective date and version number.
- Decisions made in chat are not complete until copied into the project memory files.
- Bug reports must include version, environment, expected result, actual result, and evidence.

## Recommended Rhythm

- Daily: short status update in the project group
- Weekly: version risk review led by the Project Manager
- Before release: test sign-off message with evidence links
