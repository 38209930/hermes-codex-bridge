# OpenClaw Team Playbook

Use these files as the default operating layer for every project team.

## Files

- `roles/`: role prompts for each team member
- `project-operating-rules.md`: project isolation and delivery rules
- `project-kickoff-message.md`: first message template for a new Feishu group

## Recommended Use

For each new project:

1. copy `projects/_template` to a new project folder
2. create a dedicated Feishu group for that project
3. send the kickoff message template to the group
4. assign one role prompt per OpenClaw role
5. keep all project memory updates inside that project's folder only
